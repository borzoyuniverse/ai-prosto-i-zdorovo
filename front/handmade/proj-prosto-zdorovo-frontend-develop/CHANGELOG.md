# CHANGELOG

Формат [работы с CHANGELOG](https://keepachangelog.com/en/1.1.0/).

1. При разработке фичи пишем изменения в секцию Unreleased, перед мерджем в мастер переносим изменения в раздел с текущей версией.
2. Новая версия ставится сверху списка, версия заключается в квадратыне скобки.
3. Для каждого пункта ставить номер истории по возможности
4. Выставлять под-разделы

Под-разделы:

- `Added` для новых фичей
- `Changed` для изменений в текущих фичах
- `Deprecated` для фичей, которые скоро будут удалены
- `Removed` для удаленных фичей
- `Fixed` для исправления багов
- `Security` для исправления уязвимостей
